Data files

The following data directories include examples and a little information about various file formats I have encountered.

  1. ascii_art_grayscale, a data directory of examples of grayscale ASCII Art files, in which images are created using printable characters.
  2. bamg, a data directory which contains examples of the geometry files that can be input to the BAMG meshing program.
  3. bezier_surface, a data directory of examples of bezier_surface files, a simple format comprising two files, suitable for storing the definition of a Bezier surface used in 3D graphics;
  4. cc, a data directory of examples of sparse matrices stored as Compressed Column (CC) files, a three-file format;
  5. cdl, a data directory of examples of CDL files, a text version of NETCDF files created by the NCDUMP program;
  6. cnf, a data directory of examples of CNF files, the DIMACS CNF file format, used to define examples for the satisfiability problem in conjunctive normal form;
  7. cc, a data directory of examples of sparse matrices stored as Compressed Row (CR) files, a three-file format;
  8. csv, a data directory which contains examples of CSV files, a flat file format of Comma Separated Values.
  9. dlap, a data directory of examples of DLAP files, a sparse matrix file format used by DLAP;
  10. dolfin_xml a data directory which contains examples of XML files that describe 3D finite element meshes as used by DOLFIN and FENICS.
  11. dsp, a data directory of examples of DSP files, a sparse matrix file format, storing just (I,J,A(I,J)), and using one-based indexing.
  12. eps, a data directory of examples of EPS files, Encapsulated PostScript file format for 2D graphics;
  13. fd1d, a data directory which contains examples of 1D FD files, two text files that can be used to describe many finite difference models;
  14. fd2d, a data directory which contains examples of 2D FD files, two text files that can be used to describe many finite difference models;
  15. fd3d, a data directory which contains examples of 3D FD files, two text files that can be used to describe many finite difference models;
  16. fem1d, a data directory of examples of 1D FEM files, a set of three text files that can be used to describe many finite element models;
  17. fem2d, a data directory of examples of 2D FEM files, a set of three text files that can be used to describe many finite element models;
  18. fem3d, a data directory of examples of 3D FEM files, a set of three text files that can be used to describe many finite element models;
  19. freefem++_mesh, a data directory which contains examples of the mesh files created by the FreeFem++ program, which use the extension ".msh".
  20. gif, a data directory of examples of GIF files, a 2D graphics file format;
  21. gpl, a data directory of examples of GPL files, a graphics file format used by GNUPLOT;
  22. graph_representation, a data directory of examples of ways of representing abstract mathematical graphs
  23. grf, a data directory of examples of GRF files, an abstract graph file format, 2D graphics;
  24. hb, a data directory of examples of HB files, the Harwell Boeing sparse matrix file format;
  25. hdf, a data directory of examples of HDF files, a general scientific data file format;
  26. jpg, a data directory of examples of JPG or JPEG files, a file format for high quality 2D graphics;
  27. medit, a data directory which contains examples of files which can be used by the MEDIT program to define a 2D or 3D mesh for use by the finite element method (FEM), using triangles, quadrilaterals, tetrahedrons or hexahedrons.
  28. met, a data directory of examples of MET files, a graphics file format used by the DISLIN plotting package of Helmut Michels.
  29. metis_graph, a data directory of examples of METIS GRAPH files, the format for graphs to be read by METIS, which can partition a graph;
  30. metis_mesh, a data directory of examples of METIS MESH files, the format for finite element meshes to be read by METIS, which can partition the elements of such a mesh;
  31. mm, a data directory of examples of MM files, the Matrix Market Sparse Matrix File Format;
  32. msh, a data directory of examples of MSH files, the native 3D mesh format used by gmsh.
  33. netcdf a data directory which contains examples of netCDF files, a format for the interchange of scientific data.
  34. obj, a data directory of examples of OBJ files, a Wavefront file format for 3D graphics;
  35. off, a data directory of examples of OFF files, a file format for 3D graphics;
  36. pbma, a data directory of examples of ASCII PBM files, ASCII Portable Bit Map 2D graphics, black and white;
  37. pbmb, a data directory of examples of binary PBM files, binary Portable Bit Map 2D graphics, black and white;
  38. pdb, a data directory of examples of PDB files, a file format used by the Protein Database;
  39. pdf, a data directory of examples of PDF files, the Portable Document Format, 2D text and graphics;
  40. pgh_mri, a data directory of examples of PGH MRI files, a format for storing brain scan data gathered by magnetic resonance imaging;
  41. pgma, a data directory of examples of ASCII PGM files, ASCII Portable Gray Map, 2D graphics;
  42. pgmb, a data directory of examples of binary PGM files, binary Portable Gray Map, 2D graphics;
  43. plc, a data directory of examples of PLC files, used to describe a piecewise linear complex, a surface that bounds a 3D region that is to be meshed.
  44. plot3d, a data directory of examples of PLOT3D files, a variety of formats used by the PLOT3D program;
  45. ply, a data directory of examples of PLY files, a format for objects defined as a collection of polygons, 3D graphics;
  46. png, a data directory of examples of PNG files, Portable Network Graphics, 2D graphics;
  47. poly, a data directory of examples of POLY files, a 2D format used by triangle for Planar Straight Line Graphs;
  48. poly_3d, a data directory of examples of 3D POLY files, a 3D format used by tetgen for a Piecewise Linear Complex;
  49. polygonal_surface, a data directory of examples of POLYGONAL SURFACE files, a simple file format comprising two files, suitable for storing the definition of a surface that is made up of a set of polygons of a fixed order, 3D graphics;
  50. pov, a data directory of examples of POV files, Persistence of Vision format, 3D graphics;
  51. ppma, a data directory of examples of ASCII PPM files, ASCII Portable Pixel Map, 2D graphics, color;
  52. ppmb, a data directory of examples of binary PPM files, binary Portable Pixel Map, 2D graphics, color;
  53. ps, a data directory of examples of PS files, PostScript files, 2D graphics;
  54. quad_mesh, a data directory of examples of QUAD_MESH files, a quadrilateral mesh in 2D.
  55. quad_mesh_order1, a data directory of examples of QUAD_MESH files, a quadrilateral mesh in 2D, with an associated data value that is constant over each element of the mesh;
  56. quad_surface, a data directory of examples of QUAD_SURFACE files, a quadrilateral mesh in 3D, used to define a surface.
  57. smf, a data directory which contains examples of SMF files. The SMF format was used by Michael Garland as an input and output format for his surface simplifying program QSLIM.
  58. sparse_cr, a data directory of examples of sparse matrices stored as Compressed Row (CR) files, a three-file format;
  59. st, a data directory of examples of Sparse Triplet (ST) files, a sparse matrix file format, storing just (I,J,A(I,J)), and using zero-based indexing.
  60. stla, a data directory of examples of ASCII STL files, ASCII Stereolithography files, 3D graphics;
  61. stlb, a data directory of examples of binary STL files, binary Stereolithography files, 3D graphics;
  62. svg, a data directory of examples of SVG files, Scalable Vector Graphics, 2D graphics, that can be included in an HTML file;
  63. tec, a data directory of examples of TEC files, Tecplot files used for storing 2D or 3D graphics with attributes;
  64. tet_mesh_tet_neighbors, a data directory of examples of TET_MESH_TET_NEIGHBOR files, description of a format for tetrahedron adjacency information in a tetrahedral mesh of 3D points;
  65. tri_surface, a data directory of examples of TRI_SURFACE files, a 3D surface defined by a collection of triangles.
  66. triangle_files, a data directory of examples of files used by the triangle and showme programs.
  67. triangulation_order3, a data directory of examples of TRIANGULATION_ORDER3 files, description of a linear triangulation of a set of 2D points, using a pair of files to list the node coordinates and the 3 nodes that make up each triangle;
  68. triangulation_order4, a data directory of examples of TRIANGULATION_ORDER4 files, description of a triangulation of a set of 2D points, using a pair of files to list the node coordinates and the 4 nodes that define each triangle (3 vertices and the centroid);
  69. triangulation_order6, a data directory of examples of TRIANGULATION_ORDER6 files, a description of a quadratic triangulation of a set of 2D points, using a pair of files to list the node coordinates and the 6 nodes that make up each triangle; Six-node triangles are used when a higher degree approximation is desired; they may also be used as isoparametric elements that model curved boundaries;
  70. vtk, a data directory which contains examples of "legacy" (old style) VTK files, a file format used by the Visual Toolkit;
  71. vts, a data directory which contains examples of VTS files, a file format used by the Visual Toolkit, which contains information associated with a curvilinear or structured grid, which can be displayed by the paraview program;
  72. vtu, a data directory of examples of XML-based VTU files, a file format used by the Visual Toolkit for unstructured grid data;
  73. vu, a data directory of examples of VU files, a file format used by the VU program;
  74. xml, a data directory of examples of XML files, a standard general datafile format.
